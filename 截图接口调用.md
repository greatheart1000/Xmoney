对，这就是一个**截图 API 服务端**。
用户调用这个接口后，把返回的图片字节流保存到本地就行。

下面给你几个最常用的客户端写法。

------

## Python 保存到本地

```python
import requests

url = "http://127.0.0.1:8999/api/screenshot"

resp = requests.get(url, timeout=10)
resp.raise_for_status()

with open("screen.jpg", "wb") as f:
    f.write(resp.content)

print("截图已保存到 screen.jpg")
```

如果你想按时间命名：

```python
import requests
from datetime import datetime

url = "http://127.0.0.1:8999/api/screenshot"

resp = requests.get(url, timeout=10)
resp.raise_for_status()

filename = f"screen_{datetime.now().strftime('%Y%m%d_%H%M%S')}.jpg"

with open(filename, "wb") as f:
    f.write(resp.content)

print(f"截图已保存到 {filename}")
```

## Python 封装成函数

```python
import requests
from datetime import datetime
from pathlib import Path

def save_screenshot(
    api_url: str = "http://127.0.0.1:8999/api/screenshot",
    save_dir: str = "./screenshots"
) -> str:
    Path(save_dir).mkdir(parents=True, exist_ok=True)

    resp = requests.get(api_url, timeout=10)
    resp.raise_for_status()

    filename = f"screen_{datetime.now().strftime('%Y%m%d_%H%M%S')}.jpg"
    file_path = Path(save_dir) / filename

    with open(file_path, "wb") as f:
        f.write(resp.content)

    return str(file_path)

if __name__ == "__main__":
    path = save_screenshot()
    print(f"截图已保存到: {path}")
```

## curl 方式

命令行直接保存：

```bash
curl http://127.0.0.1:8999/api/screenshot --output screen.jpg
```

按时间命名的话，在 PowerShell 里可以这样：

```powershell
$ts = Get-Date -Format "yyyyMMdd_HHmmss"
curl http://127.0.0.1:8999/api/screenshot --output "screen_$ts.jpg"
```

## JavaScript / Node.js 保存

```javascript
const fs = require("fs");
const http = require("http");

const url = "http://127.0.0.1:8999/api/screenshot";
const file = fs.createWriteStream("screen.jpg");

http.get(url, (res) => {
  if (res.statusCode !== 200) {
    console.error("请求失败:", res.statusCode);
    return;
  }

  res.pipe(file);

  file.on("finish", () => {
    file.close();
    console.log("截图已保存到 screen.jpg");
  });
}).on("error", (err) => {
  console.error("请求出错:", err.message);
});
```

## 你这个接口返回的是 JPEG

因为你服务端这里写的是：

```python
img.save(img_byte_arr, format='JPEG', quality=85)
return Response(content=image_bytes, media_type="image/jpeg")
```

所以客户端保存时，最好用：

- `.jpg`
- `.jpeg`

不要保存成 `.png`，不然格式和后缀不一致。

## 更稳一点的 Python 版本

顺手校验一下返回类型：

```python
import requests

url = "http://127.0.0.1:8999/api/screenshot"
resp = requests.get(url, timeout=10)
resp.raise_for_status()

content_type = resp.headers.get("Content-Type", "")
if "image/jpeg" not in content_type:
    raise ValueError(f"返回的不是 JPEG 图片，而是: {content_type}")

with open("screen.jpg", "wb") as f:
    f.write(resp.content)

print("保存成功")
```

## 如果你后面要接到 AI 分析里

其实可以不落盘，直接拿字节流继续传：

```python
import requests

resp = requests.get("http://127.0.0.1:8999/api/screenshot", timeout=10)
resp.raise_for_status()

image_bytes = resp.content

# 这里可以直接发给 OCR / AI / 另一个接口
print(f"拿到图片字节长度: {len(image_bytes)}")
```

## 一句话总结

你的服务端负责：

- 截图
- 返回图片字节流

客户端负责：

- 调接口
- 把 `resp.content` 写入本地文件

最简单可用版就是这几行：

```python
import requests
resp = requests.get("http://127.0.0.1:8999/api/screenshot")
with open("screen.jpg", "wb") as f:
    f.write(resp.content)
```
